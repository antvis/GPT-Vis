Flow Diagram

流程图，用于直观地表示过程或系统的步骤和决策点。它展示了从开始到结束的整个流程。每个节点代表一个特定的步骤或决策点，边则表示步骤之间的顺序和关系。只有在有分支表意的情况下，边才需要命名。

Use Cases

  • 适用于需要展示线性流程或步骤的场景。
  • 规划和跟踪项目进度，明确任务的先后顺序和依赖关系。
  • 构建决策树，展示不同决策点和路径的场景。

Visual Examples

性能诊断流程图，展示从 I/O 检查到问题定位和解决方案的决策流程。

用户注册流程（dark 主题）：用户访问注册页面，填写注册表单并提交，系统验证用户信息，创建账户或提示修改，发送验证邮件，用户验证后注册成功。

订单配送流程：客户下单、系统生成订单、仓库拣货、仓库打包、物流配送、客户收货。

Options

Configuration Options

PropertyTypeDescription
typerequired图表的类型，必填，文本类型，值必须为 "flow-diagram"。
datarequired图表的数据，必填，对象类型，包含 nodes 和 edges。
data.nodesrequired节点数组，每个节点表示一个步骤，必填，数组对象类型。
data.nodes.namerequired节点的名称，必须唯一，必填，文本类型。
data.edgesrequired边数组，每条边表示两个节点之间的关系，必填，数组对象类型。
data.edges.sourcerequired边的起始节点名称，指向节点的 name 属性，必填，文本类型。
data.edges.targetrequired边的目标节点名称，指向节点的 name 属性，必填，文本类型。
data.edges.nameoptional边的名称，用于标识边（分支场景），选填，文本类型。
titleoptional图表标题，选填，文本类型。
themeoptional图表主题，选填，文本类型，可选值为 "default" | "dark" | "academy"，默认值为 "default"。
styleoptional图表样式，选填，对象类型。
